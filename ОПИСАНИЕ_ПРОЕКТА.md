# МакраМир — Описание дипломного проекта

Сайт интернет-магазина по продаже шнуров для плетения гамаков и изделий ручной работы.

---

## Структура проекта

```
diploma/
│
├── index.html          ← главная страница (HTML-разметка)
│
├── css/
│   └── style.css       ← все стили и оформление сайта
│
├── js/
│   └── main.js         ← вся логика и интерактивность
│
├── image/              ← фотографии товаров
│   ├── 6N4A8869.jpg
│   ├── 6N4A8948.jpg
│   └── ... (48 фото)
│
└── ОПИСАНИЕ_ПРОЕКТА.md ← этот файл
```

---

## Как открыть сайт

Просто дважды кликни по файлу `index.html` — он откроется в браузере.  
Интернет нужен только для загрузки шрифтов Google Fonts (навигация и текст).  
Все остальное работает локально без интернета.

## Описание файлов

### index.html — разметка страницы

HTML-файл описывает **структуру** сайта: какие блоки есть, в каком порядке расположены.  
Сам по себе не задаёт цвета или поведение — для этого подключены CSS и JS.

Файл разбит на секции, каждая обозначена комментарием:
```html
<!-- НАВИГАЦИЯ -->
<!-- ПОИСК -->
<!-- КОРЗИНА -->
<!-- ГЛАВНЫЙ ЭКРАН (HERO) -->
<!-- ПРЕИМУЩЕСТВА -->
<!-- КАТАЛОГ -->
<!-- О НАС -->
<!-- ГАЛЕРЕЯ -->
<!-- ПРОМО-БАННЕР -->
<!-- ДОСТАВКА -->
<!-- КОНТАКТЫ -->
<!-- ПОДВАЛ (FOOTER) -->
```

В конце файла — `<script src="js/main.js"></script>`.  
Скрипт подключается **в конце** — чтобы HTML успел загрузиться раньше, чем JavaScript начнёт с ним работать.

---

### css/style.css — оформление

CSS-файл задаёт **внешний вид**: цвета, шрифты, отступы, анимации, адаптивность.

**Ключевые приёмы, которые использованы:**

**CSS-переменные** (в начале файла, блок `:root`):
```css
:root {
  --c-primary: #5C3524;   /* основной цвет — тёмно-коричневый */
  --c-accent:  #C68B2A;   /* акцент — золотистый */
}
```
Вместо того чтобы каждый раз писать `#5C3524`, пишем `var(--c-primary)`.  
Хочешь изменить цвет сайта — меняешь только в одном месте.

**CSS Grid** — для сетки товаров:
```css
.prod-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(285px, 1fr));
}
```
Автоматически расставляет карточки: сколько помещается в строку — столько и будет.

**CSS Flexbox** — для навигации, карточек, кнопок:
```css
.nav-in {
  display: flex;
  align-items: center;
  justify-content: space-between;
}
```

**Медиазапросы** — адаптивность:
```css
@media (max-width: 768px) {
  .nav-links { display: none; }  /* скрываем ссылки на мобильном */
  .burger    { display: flex; }  /* показываем кнопку-гамбургер */
}
```

**@keyframes** — CSS-анимации:
```css
@keyframes fadeUp {
  from { opacity: 0; transform: translateY(40px); }
  to   { opacity: 1; transform: translateY(0); }
}
```

### js/main.js — логика и интерактивность

JavaScript-файл отвечает за **поведение** сайта: что происходит при нажатии кнопок, как работает поиск и корзина.

**Данные товаров** хранятся в массиве `PRODUCTS`:
```javascript
const PRODUCTS = [
  {
    id: 1,
    name: 'Шнур гамаковый Мокко, 5 мм',
    price: 5.5,
    // ...
  },
  // ...
];
```
Изменить товар — просто отредактировать объект в массиве.

**Корзина** сохраняется в `localStorage`:
```javascript
// Сохранить
localStorage.setItem('mkm_cart', JSON.stringify(cart));

// Загрузить
let cart = JSON.parse(localStorage.getItem('mkm_cart') || '[]');
```
`localStorage` — это хранилище браузера. Данные не пропадают при закрытии вкладки.

**IntersectionObserver** — анимации при прокрутке:
```javascript
const observer = new IntersectionObserver(function(entries) {
  entries.forEach(function(entry) {
    if (entry.isIntersecting) {       // элемент появился на экране?
      entry.target.classList.add('on'); // добавляем класс → CSS запускает анимацию
    }
  });
});
```

**Поиск** работает без перезагрузки страницы:
```javascript
searchInput.addEventListener('input', function() {
  const found = PRODUCTS.filter(function(p) {
    return p.name.toLowerCase().includes(query);
  });
  // показываем результаты...
});
```

## Технологии

# ВОПРОСЫ ЖЮРИ И ОТВЕТЫ

---

### «Расскажи о своём проекте. Что это такое?»

*Я разработал сайт интернет-магазина «МакраМир» для продажи полиэфирных шнуров для плетения гамаков и подвесных кресел. Сайт позволяет просматривать каталог товаров, фильтровать их по категориям, добавлять в корзину, осуществлять поиск, а также связаться с магазином через форму. Всё сделано на чистом HTML, CSS и JavaScript — без готовых фреймворков.*

---

### «Почему именно такой дизайн, такие цвета?»

*Я изучил несколько сайтов схожей тематики и заметил: магазины изделий ручной работы и натуральных материалов чаще всего используют тёплые, земляные тона. Коричневый и золотистый вызывают ассоциации с природой и натуральностью — это соответствует самому товару. Светлый кремовый фон делает страницу визуально лёгкой, не отвлекает от товаров.*

---

### «Зачем вы разделили код на три файла?»

*Разделение кода на HTML, CSS и JavaScript — это стандартная практика в веб-разработке. Принцип называется «разделение ответственности»: HTML описывает структуру, CSS — внешний вид, JavaScript — поведение. Если нужно изменить цвет кнопки — открываю style.css, если логику корзины — main.js, если структуру секции — index.html. При едином файле всё превращается в кашу и сложно ориентироваться.*

---

### «Что такое адаптивная вёрстка и как ты её реализовал?»

*Адаптивная вёрстка — это когда сайт корректно отображается на любом устройстве: компьютере, планшете, телефоне. Реализовал через медиазапросы `@media` в CSS. Например, при ширине экрана меньше 768 пикселей — скрываю ссылки навигации и показываю кнопку-гамбургер, сетка товаров перестраивается в одну колонку. Ещё использую `grid-template-columns: repeat(auto-fill, minmax(285px, 1fr))` — это CSS Grid, который сам подбирает количество колонок под ширину экрана.*

---

### «Как работает корзина? Где хранятся данные?»

*Корзина работает полностью на стороне браузера. Когда пользователь нажимает «В корзину», JavaScript добавляет товар в массив `cart`. Чтобы данные не пропадали при обновлении страницы, я сохраняю этот массив в `localStorage` — это встроенное хранилище браузера, которое не очищается при закрытии вкладки. При загрузке страницы корзина восстанавливается из `localStorage`. Сумма пересчитывается автоматически с помощью метода `reduce`.*

---

### «Как устроен поиск?»

*Поиск работает без обращения к серверу — прямо в браузере. Все товары хранятся в JavaScript-массиве `PRODUCTS`. При каждом нажатии клавиши в поле поиска вызывается метод `filter()`, который проверяет: содержит ли название или описание товара введённые символы. Найденные товары сразу отображаются как карточки. Это называется «поиск на стороне клиента» — он мгновенный, потому что нет запроса к серверу.*

---

### «Что такое localStorage?»

*`localStorage` — это хранилище данных, встроенное в каждый браузер. Туда можно записывать данные в виде строки и читать их обратно. В отличие от обычных переменных, данные в `localStorage` сохраняются даже после перезагрузки страницы или закрытия браузера. Я использую его для корзины: когда пользователь добавляет товар — пишу в хранилище, при загрузке — читаю из него. Данные хранятся в формате JSON.*

---

### «Что такое CSS-переменные и зачем они нужны?»

*CSS-переменные — это именованные значения, которые можно использовать повторно. В начале style.css я описываю все цвета один раз в блоке `:root`, например `--c-primary: #5C3524`. Потом везде в коде пишу `var(--c-primary)` вместо конкретного значения. Если клиент попросит изменить основной цвет — меняю в одном месте, и это автоматически применяется по всему сайту. Без переменных пришлось бы менять сотни строк.*

---

### «Что такое CSS Grid и CSS Flexbox? В чём разница?»

*Оба используются для раскладки элементов. Flexbox работает в одном направлении — либо в строку, либо в колонку. Я использую его для навигации, карточек с иконками, строк корзины. CSS Grid работает в двух направлениях сразу — строки и колонки. Я использую его для сетки товаров и раздела «О нас». Грубо говоря: Flexbox — для простых строк/колонок, Grid — для полноценных макетов.*

---

### «Что такое IntersectionObserver?»

*Это браузерный API, который следит за тем, виден ли элемент в окне браузера. Я его использую для анимаций при прокрутке: элементы изначально невидимы (CSS класс `.anim` ставит `opacity: 0`). Когда пользователь прокручивает страницу и блок появляется на экране — Observer замечает это и добавляет класс `.on`, который через CSS делает элемент видимым с анимацией «выезда снизу». Без этого пришлось бы постоянно слушать событие `scroll` — это менее производительно.*

---

### «Почему нет базы данных и серверной части?»

*Задача дипломного проекта — разработать клиентскую часть (фронтенд). Серверная часть (бэкенд) — это отдельная дисциплина, требующая знания таких технологий как Node.js, PHP или Python, а также SQL. Реализованный функционал — каталог, поиск, корзина — работает на стороне браузера. В реальном проекте форма отправляла бы данные на сервер, а товары загружались бы из базы данных. Для учебных целей данные хранятся в JavaScript-массиве, корзина — в localStorage.*

---

### «Использовал ли ты какие-то готовые библиотеки?»

*Нет. Весь код написан самостоятельно на чистом JavaScript и CSS, без библиотек типа jQuery или Bootstrap, без CSS-фреймворков. Для шрифтов подключил Google Fonts — это бесплатный сервис Google, одна строчка в HTML. Для карты использую OpenStreetMap через `<iframe>` — тоже бесплатно и не требует регистрации. Отсутствие библиотек — осознанное решение: код проще понять, нет лишних зависимостей, страница загружается быстрее.*

---

### «Как работают анимации на главном экране?»

*При загрузке страницы заголовок, подзаголовок и кнопки изначально прозрачны (`opacity: 0`). В CSS для них прописана анимация `@keyframes fadeUp` с задержкой: заголовок появляется через 0.2 секунды, подзаголовок — через 0.45, кнопки — через 0.65. Это создаёт эффект последовательного появления. Фоновое фото при загрузке немного уменьшается (`scale(1.06 → 1.0)`) — это создаёт лёгкий эффект «приближения». Эффект запускается через JavaScript: при загрузке страницы добавляется CSS-класс `.go`.*

---

### «Что такое семантическая разметка?»

*Семантические теги — это HTML-теги, которые описывают смысл содержимого, а не только внешний вид. Например, `<nav>` явно говорит браузеру и поисковику «здесь навигация», `<section>` — «это отдельный раздел», `<footer>` — «подвал страницы», `<aside>` — «дополнительный блок» (в моём случае корзина). Это важно для SEO и для людей, которые используют скринридеры (программы для незрячих).*

---

### «Что такое `<iframe>` и как ты использовал его?»

*`<iframe>` — это тег для встраивания одной веб-страницы внутрь другой. Я использовал его для карты в разделе «Контакты»: OpenStreetMap предоставляет готовый URL для встраивания, я вставил его в `src` тега `<iframe>`. Так карта отображается прямо на моём сайте без необходимости API-ключей и оплаты, в отличие от Google Maps.*

---

### «Как работает hover-эффект смены фотографии на карточке товара?»

*У каждой карточки две фотографии: основная и вторая. Они лежат одна поверх другой через `position: absolute`. По умолчанию вторая фотография прозрачна (`opacity: 0`). При наведении курсора (CSS-псевдокласс `:hover`) первая фотография становится прозрачной, а вторая — видимой. `transition` делает смену плавной за 0.55 секунды. Одновременно обе фотографии немного увеличиваются (`scale(1.06)`) — это создаёт эффект «приближения».*
